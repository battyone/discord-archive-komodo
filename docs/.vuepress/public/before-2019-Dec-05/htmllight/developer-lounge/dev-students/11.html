yes, the end goal is to make a Go language code which works the same way as the faucet CC code works in C language.
There are 2 types of CC code examples in FSM branch of jl777/komodo repo.
1. The CC code written in C, which is compiled with the komodod and is available from within of komodod. 2. Same like 1, written in C, but compiles to an independent dynamic library, which sites right next to the komodod binary and is the komodod calls the RPC/API/methods from this library. It is the dynamic CC creation method, which doesn't need recompiling the whole komodod and only the CC code part. Makes the devs' life much better to only care about the CC code and also helps distribution of the CC to end users in updates when/if needed.
You should check both examples of this inside compiled komodod CC code, and the dynamic linkable CC code which compiles to library file.
What James asked is the 2nd option.
Just understand the 2nd way, dynamically linkable CC code lib, and try to make that in Go language
