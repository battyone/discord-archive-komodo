dimxy 27-Oct-19 07:26 PM
This "error": "some inputs signed not with -pubkey=pk" means that token creator wallet has other privkey(s) in addition to -pubkey=pk. It is recommended that the tokencreator has only one pk in the wallet. Actually this check prevents that a token creator creates a token and puts some pubkey in the token opreturn data for which he does not have the privkey (making the token users believe that the token is really created by the one whose pubkey is in the token data)
