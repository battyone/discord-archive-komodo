Mylo 08-Oct-18 02:36 PM
haha ok.
git clone https://github.com/imylomylo/docker-komodo-smartcontracts.git cd docker-komodo-smartcontracts/src make run
You'll be dropped into a container. That repo has two directories. 1. src/ has the Makefile to simplify repetitive commands it's just docker commands. 2. node/ directory. this gets mounted at ~/.komodo inside the container. Your username in the container is user and the password is pass. You have sudo access so you can sudo su - and then look at network stuff like ifconfig or netstat -ptan etc.
You can screen -S kmd and type komodod & and it will start in regtest mode because that is the setting in ~/.komodo/komodo.conf. ^a d will detach from that session. alternatively you can ssh to your container. in regtest mode you can do stuff like komodo-cli generate 5 and will mine 5 blocks. Enough about regtest mode. Kill the container by exit and you get dropped back to your shell. To create your own AC (network), comment out regtest=1 in node/komodo.conf (if you start a KMD daemon now it will connect to mainnet). What you want is your own network in docker. The Makefile has a make target called acnet. So now you can: make acnet then make runacnet for 1 container. Then do it again for another container on the same docker network make acnet <-- probably in another terminal window. If you sudo ifconfig in each, you get their ip address. SSH server is not started by default atm. so in each you will need to /etc/init.d/ssh start and then you can ssh to their docker network addresses, something like 172.19.0.2 and 172.19.0.3 from memory will be the first addresses on DHCP range. Create your AC in each of those.
The dockerfile has been made in a way that you can access ~/.komodo back in your normal working environment (the cloned github repo) - WARNING
