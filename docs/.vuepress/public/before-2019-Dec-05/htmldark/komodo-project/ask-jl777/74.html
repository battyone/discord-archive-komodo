jl777c 30-Jun-19 11:04 AM
SPV wallets allow lite wallets to only download header data (and a little bit more per utxo)
hyperDEX 30-Jun-19 11:04 AM
Lite clients are lighter now
jl777c 30-Jun-19 11:04 AM
but for a blockchain with a million blocks, that is a million headers (2GB)
which isnt so lite... non-equihash coins have 80 byte headers so the problem is really bad for equihash coins
that means for a normal SPV wallet, you would need to sync 2GB+ of headers before you can validate the utxo, and each utxo needs merkleproofs on top
with nSPV, on average each utxo needs 20kb of headers and the usual merkleproofs
so a wallet with 10 utxo would need 200kb vs 2GB of headers
10000x difference in bandwidth and it is so small, it doesnt even need to be stored locally
also, the issue of attacks is dramatically mitigated as only a single honest node is needed and all dishonest nodes are immediately found out as the notarizations cant be forged
this is called superlight clients
it will basically achieve the same performance level of electrum servers, without any servers
so it solves the scalability issue quite completely
