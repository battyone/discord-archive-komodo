Decker 02-Dec-18 08:40 AM
Important note to all Komodo-Qt (KomodoOcean) users according to new features coming on December 15 - https://komodoplatform.com/upgrade-guide-dec-15/ . As we are all know coming update brings a lot of new features, such as sapling, VerusHash algorithm and others. This is a massive update that includes changes to over 500 different source files and adding new dependencies. Komodo-Qt will follow original komodod branch and will updated too as soon as possible, but this hard work unfortunatelly will not be finished to 15 Dec (!), in this way all komodo-qt users should temporarily switch to use original komodo daemon (komodod) or Agama, until update for komodo-qt will be released. All your funds are safe, just backup wallet.dat and continue to use your wallet.dat from Komodo-Qt with komodod or Agama, or export your privkeys from Komodo-Qt using z_exportwallet and import them into komodod with z_importwallet. We will keep you informed about current state of Komodo-Qt and coming updates, but keep in mind that current version of Komodo-Qt will be incompatible with new changes after sapling is activated, so, you should temporary stop using it and switch to other wallet.

wookie 13-Jan-19 05:21 AM
What's the difference between using Agama vs QT? Is QT more user friendly?
Decker 13-Jan-19 09:27 AM
Qt has a well-known Bitcoin Core interface with it's own features, like advanced Coin Control Feautures (you can select specify utxos for sending or combining), it's not an GUI shell on komodod, like Agama in native mode, it has "daemon inside", so, it's "standalone wallet". And it fully written in C/C++ and Agama uses NodeJS, so, it should be little bit faster. What to use Agama or Komodo-Qt - it's a matter of personal preferences. For me - Komodo-Qt it's a main wallet, but when i need SPV (Electrum), i'm using Agama.
