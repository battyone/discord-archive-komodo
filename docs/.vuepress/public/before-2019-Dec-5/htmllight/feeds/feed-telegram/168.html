[telegram] <Regnar> So Jumblr automates the process, check out the white paper, page 79. It performs actions randomly to optimize anonymity, and will cluster all jumblr users transactions of same amounts (standard size transactions) around the same blocks as well (multiples of 10) this protects against timing attacks (analysis of transaction outputs and input times to follow transactions) because user blend the timing of their transactions together with jumblr. Also because they are equal sized lots (you jumblr 120kmd is lots 100, 10, 10) you prevent analysis by TX size (like I mentioned earlier) Jumblr will also use multiple secret addresses to further obfuscate these transactions, and still time the movements randomly but grouped with others. So this whole process is automatic and is more secure than just moving through a secret address.
