dimxy 29-Apr-19 11:20 AM
@blackjok3r actually this call ./komodo-cli -ac_name=CFEKDRAGON tokencreate token 1 "a test token" "some non fungible data stuff" creates not a NFT token, but a fungible token. This is because "some non fungible data stuff" was not parsed (as it should be in hex format like "ED0102030405..." with 1st byte is the eval code of cc responsble for NFT validation) and tokencreate switched to fungible token (with no such param) So it was actually a fungible token. For it there is no amount=1 sat limit and it was actually 10000000 sat token created (as amount param is in coins, not satoshis). So for such a token we could indeed import 1 or 10 sat or any amount of it. For every such import a new import token would be created in the destination chain with a new tokenid and a reference to the source token. But we cannot import more than the initial total amount and before the each import of some token amount the same amount is burned in the source chain. To prevent this issue with the param I added a check to tokencreate which would output an error if NFT data hex param was not parsed successfully.
