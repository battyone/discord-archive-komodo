This SDK will not only provide the blockchain_interface layer - the 2nd essential part is the gaming layer (that is particularily the UI/gfx layer - but not limited to this) - which provides devs a basic game skeleton with the game loop, input/output functions, drawing functionality, audio interface, etc. - lets say someone wants to build a fancy 2 player PacMan GUI game that stores gameplay records/highscores, etc. to the blockchain: Komodos new Gamign SDK will even offer a command like antara_sdk create SFML pacman (just a pseudo example) which would provide you a folder pacman with ALL required SFML libs/bindings, the games main function (gameloop,...) etc. Devs would be able to just start building their game by adding the required assets, game logic, blockchain interaction calls, etc. and voila!
