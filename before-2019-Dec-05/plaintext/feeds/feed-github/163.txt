[KomodoPlatform/atomicDEX-API] New comment on issue #514: \[Feature request\] redirect logs to a file
I'd like to add some backtround here: as far as I know, Dart does not have first-class support for spawning the MM process with stdout redirected to a file (in comparison Rust supports stdout redirection with `Stdio::from_raw_fd`), and redirecting via a shell is less reliable, particularly when JSON configuration is passed via the command-line.
So an option for MM to write the logs directly to a file might be useful.

I plan to work on this later today.
