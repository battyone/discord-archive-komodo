cipi 27-Sep-18 12:51 PM
@grewalsatinder @CHMEX yes, i configured the wallets on the electrum servers to listen to .onion, so almost every coin on the electrums also offers .onion i used this config for KMD
server=1 txindex=1 disablewallet=1 rpcuser=eleckmd rpcpassword=whatever onion=127.0.0.1:9050 listen=1 listenonion=1 bind=127.0.0.1
i used the onion-option instead of proxy, so that the nodes can connect to other onions, but also directly to normal nodes, out of performance reasons (tor is simply much slower then normal internet) listenonion=1 can only be used when listen is also 1 (if you set listen to 0, listenonion=1 has no effect), but because i don't wanted to let this nodes listen to normal connections (only to onions) i set both to 1 and also used bind=127.0.0.1 tor must be installed (sudo apt-get install tor) and the torrc file must contain the parameter ControlPort 9051 and the user that runs the wallet must be in the group debian-tor (usermod -G debian-tor -a username)
