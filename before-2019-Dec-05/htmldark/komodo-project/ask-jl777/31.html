gcharang 27-Nov-18 08:09 AM
When people refer to smart contracts, like in ETH, all the nodes+miners know a specific programming language(Ex: solidity). The contracts are written on the blockchain using the programming language and as all nodes know the language, they will evaluate them. In Komodo's architecture, there is no such thing as solidity. The code for the contracts is already in the daemon. The only data that gets written on the blockchain is a reference to the contract and any other inputs it requires. So theoretically, the code in the daemon can be written in any language. This also gives us the possibility for a modular architecture, where you specify the contracts you want to run on your blockchain when compiling your daemon and only those contracts are usable on the chain created using it. Currently all the contracts get added to the daemon by default when you use it for assetchains but this modular functionality is planned for the future when there are more contracts
