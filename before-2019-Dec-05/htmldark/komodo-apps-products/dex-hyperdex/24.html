lukechilds 30-Sep-18 12:03 AM
Regarding mmv2: There are certain features we need to build a reliable UI that we don't have in mmv1. mmv2 is pretty much a direct rewrite of mmv1 in Rust with better support for concurrency, along with thread/memory safety and a few other things Rust gives us for free. So it won't automatically fix all of our problems as soon as the first mmv2 version is released, although it should work much better than mmv1. It will likely still have some of the mmv1 warts due to it being a direct port. But it should mean we'll have a much more solid foundation to work with the mm dev team to get bugs fixed and new API methods and stuff added that we need that just aren't feasible with the current mmv1 codebase. TL;DR mmv2 isn't a silver bullet that will automatically solve all our problems. But it is a required next step before we can go much further with HyperDEX development. (edited)
