Alright 23-Nov-18 12:00 PM
You can earn LABS by finding bugs in smart contracts, staking, mining or running a notary. We set up a system to where we can add contracts on the fly without the need for a hard fork. A contract dev can spin up a chain(only cost is notarization cost), people can test this chain, if a bug is found they will make a bounty claim. If enough notaries sign off on this bounty claim, they get the coins
So contract dev can start a chain and have people test it for basically no cost the them
here's a elevator pitch I typed up previously
we'll have a website up with more info next week
Purpose of STAKED is to give crypto condition contract devs a place to have their contracts tested. There is a problem in that a coin dev cannot just implement the first iteration of a new contract on a production coin as each bug could mean a hard fork.(think KMDICE the last couple weeks) What STAKED allows a dev to do is spin up a STAKED chain that is fungible with all other STAKED chains. They can have custom consensus rules or contracts. Testers can then be rewarded by finding bugs in these contracts. If there is a bug in the contract, they simply kill the chain, update the contract and start another chain. STAKED will be the only asset chain with "opt-in contracts", meaning contracts are not compulsory consensus rules. In order to implement a contract on a typical assetchain, all nodes must update. On STAKED however, you simply add the contract to a new chain. If a user wants to use the contract, they sync the chain and transfer coins to it. If user isn't interested in the contract, they can simply ignore it.
replace STAKED with KMDLabs
