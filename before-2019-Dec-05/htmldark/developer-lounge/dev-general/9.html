@Mylo @artemciy_dice Thanks for the suggestions. I found a good-enough fix for now. It's not recursive, but it'll do. The problem turned out to be deriving from the asynchronous nature of the calls to the database. I was getting the final callback event called multiple times. Used something that I've never used before, called a semaphore -- a variable where teh value goes up with each call, and goes down once the data is processed. Final event can only call if the semaphore's value has return to zero. Almost done with this project! Thank you for the help.
