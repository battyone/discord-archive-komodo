blackjok3r 31-Mar-19 07:42 AM
I have worked out how to do notary pay for 0BR chains. Turned out to be extremly simple.
Need to tally notarypay in connect tip, and dissconnect tip. Each time a notary pay coinbase is paid we add to total, when its been reorged we deduct total. Write the total to a file each time this happens. Load the file on daemon init, and delete it if the daemon has been instructed to reindex. There is a burn address that uses a private key of all zeros. RPC will be created that checks if the amount paid is greater than the amount burnt. Iguana will poll this RPC at the start of a notarization round to ask if it can notarize or not.
In the event of a somone forgetting to top up the burn address and notarizations have stopped, we can use the notary backup import to get funds to the chain. I can also add the notary pay amounts to the notarizations DB, and there can be a backup that loops over the notarization DB and tallies all the amounts paid, in case of the file being corrupted. Not sure if this is even needed at all, guess feild test will tell us.
